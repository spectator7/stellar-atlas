# 天象志发展路线图

> 当前阶段：`v0.2` 88 星座内容基础版
> 项目目标：建设一部兼具专业星图、文化叙事和实用观测能力的中文在线星空图鉴。

## 一、现状与方向

当前版本已经覆盖 88 个现代星座，具备动态星空、真实主要星位与连线、亮星与深空目标档案、双重文化叙事、观测提示、搜索、筛选和收藏功能。内容基础已经完整，下一步将继续提升天文计算能力与专题深度。

后续更新的重点不只是增加页面数量，而是建立一套可持续维护的内容、天文数据和功能架构，使项目逐步从展示原型发展为真正可使用、可查阅、可共同维护的星空图鉴。

## 二、建设原则

1. **准确优先**：天体位置、星等、距离和观测信息应来自可靠数据，并记录来源与核对日期。
2. **文化分层**：现代星座、中国传统星官和其他文明的星空体系分别说明，不将它们简单视为一一对应关系。
3. **渐进升级**：先完善内容和数据结构，再引入复杂框架、实时计算或后端服务。
4. **移动优先**：星图、筛选和观测工具必须适合手机在户外使用。
5. **开放维护**：内容应拆分成独立数据和文章文件，方便通过 GitHub 持续修订与投稿。
6. **保护隐私**：定位功能保持可选，并支持手动选择城市或输入经纬度。

## 四、v0.2：八十八星座内容基础版

### 目标

一次性建立完整的 88 个现代星座目录，并为每个星座提供真实主要星位与连线、亮星、深空目标、观测路线、中国星官说明和西方神话或命名史。常见星座继续保持更丰富的观测内容，冷门与极南天星座也不保留空白条目。

### 首批重点深化星座

| 季节 | 星座 |
| --- | --- |
| 冬季 | 金牛座、双子座、大犬座、小犬座、御夫座 |
| 春季 | 狮子座、室女座、牧夫座、后发座 |
| 夏季 | 天鹰座、人马座、蛇夫座、武仙座 |
| 秋季 | 仙女座、英仙座、仙王座、双鱼座 |

### 每个星座的内容模板

1. 中文名、拉丁名、国际缩写和名称含义。
2. 最佳观测季节、时间、方向和适用纬度。
3. 主要恒星的名称、视星等、距离和光谱类型。
4. 真实恒星位置、传统连线和现代星座边界。
5. 代表星云、星团、星系和其他深空天体。
6. 中国传统星官及其与现代星座边界的差异。
7. 希腊及其他文明中的相关故事。
8. 裸眼、双筒望远镜和小型望远镜观测步骤。
9. 数据来源、参考资料和最后核对日期。

### 当前完成情况

- 已收录国际天文学联合会体系中的 88 个现代星座。
- 88 个星座均具备真实主要星位与连线、主要恒星和观测说明。
- 88 个星座均具备中国星官说明和西方神话或命名史。
- 已展示亮星、深空目标、数据来源和最后核对日期。
- 已支持按季节、可见范围、名称和收藏状态筛选。
- 数据文件具备自动完整性检查脚本。

## 五、v0.3：真实可探索星图

### 功能范围

- 鼠标、触摸板和手机手势缩放。
- 拖动浏览相邻天区。
- 赤经赤纬坐标网格。
- 恒星亮度、颜色和大小差异。
- 星名、星座连线、边界和深空天体图层。
- 按极限星等筛选可见恒星。
- 点击恒星或深空天体打开详细资料。
- 在相邻星座之间连续移动，而不是切换独立示意图。
- 支持北半球、南半球和不同季节视图。

### 技术原则

天体坐标转换、岁差、升落和天空投影应采用成熟的天文计算库，不自行编写核心天文算法。视觉层可以继续使用 Canvas 或 WebGL，内容数据与绘制逻辑保持分离。

### 验收标准

- 星图可以稳定缩放和拖动。
- 星体位置来自统一的赤经赤纬数据。
- 图层开关不会造成明显卡顿或布局变化。
- 手机端连续操作保持流畅。

## 六、v0.4：今晚星空与观测助手

### 功能范围

- 手动选择城市或输入经纬度。
- 可选读取设备位置。
- 设置日期、当地时间和时区。
- 显示当前可见星座、方位和高度。
- 显示主要天体的升起、过中天和落下时间。
- 提供月相、月光影响和推荐观测窗口。
- 收录流星雨、行星合月等月度天象。
- 接入天气、云量或光污染信息时提供数据来源说明。
- 生成可复制或分享的观测清单。

### 验收标准

- 不授权定位也能完整使用。
- 时间和位置变化后，结果可以即时更新。
- 观测结果明确标注日期、地点和时区。
- 对无法保证精度的信息给出清晰说明。

## 七、v0.5：星空故事资料库

### 中国星空专题

- 三垣二十八宿概览。
- 北斗与古代时令。

- 紫微垣与古代天文制度。
- 中国古代星图、诗词和观测史。

### 世界星空专题

- 希腊英雄星座关系图。
- 阿拉伯恒星命名传统。
- 巴比伦、埃及和波利尼西亚航海星空。
- 不同文明对同一片天区的比较。

### 内容规范

- 区分史料记载、后世传说和现代改编。
- 每篇专题附参考资料、相关星座和年代信息。
- 避免把单一版本写成唯一正确的神话。
- 使用原创概述，不大段复制受版权保护的内容。

## 九、v1.0：完整中文星空图鉴

`v1.0` 的目标不是简单收齐 88 个星座名称，而是形成一套完整、可检索、可观测、可核查的内容体系。

### 建议发布标准

- 收录国际天文学联合会划分的 88 个现代星座。
- 收录主要亮星及常见中文、国际名称。
- 收录适合业余观测的代表深空天体。
- 提供真实天空位置和日期时间计算。
- 建立中国传统星官专题及跨文化故事资料库。
- 主要内容均有来源和最后核对日期。
- 完成桌面、平板、手机和无障碍测试。

## 十、建议的数据与内容结构

短期仍可使用原生 HTML、CSS 和 JavaScript，但应逐步将页面内容拆分出来：

```text
data/
  constellations/
  stars/
  deep-sky/
  observing/
content/
  stories/
  guides/
  history/
assets/
  images/
  charts/
```

进入真实星图阶段后，再迁移到 `Vite + TypeScript`。在内容和数据模型稳定之前，不优先建设账号、评论、私信或复杂后端。

## 十一、发布节奏

建议每两周发布一个小版本，每次更新至少完成以下四项中的三项：

- 新增 3–5 个完整星座条目。
- 发布 1 篇星空文化专题。
- 完成 1 项星图或观测交互改进。
- 修正一批数据、无障碍或移动端问题。

每次发布在 GitHub Release 或更新日志中记录：新增内容、功能变化、数据修订、已知问题和下一版本目标。
