# 天象志发展路线图

> 当前阶段：`v0.6` 观测工作流、离线体验与开放维护基础建设
> 项目目标：建设一部兼具专业星图、文化叙事和实用观测能力的中文在线星空图鉴。
> `v0.7` 至 `v1.0` 的详细执行计划见 [V1_PLAN.md](V1_PLAN.md)。

## 一、现状与方向

当前版本已经覆盖 88 个现代星座，并完成可缩放、可拖动的连续天球星图。日期、时间、地点、设备、月光、行星升落、月度事件、最佳观测时刻与可选天气已经进入“今晚星空”观测助手；12 篇独立专题也已形成时间线、关系图、跨文明比较与来源体系。网站现已具备离线缓存、安装入口、站点地图、结构化数据和自动校验。下一步重点是持续事实核查、投稿规范、链接巡检、真机回归和内容深度。

后续更新的重点不只是增加页面数量，而是建立一套可持续维护的内容、天文数据和功能架构，使项目逐步从展示原型发展为真正可使用、可查阅、可共同维护的星空图鉴。

## 二、建设原则

1. **准确优先**：天体位置、星等、距离和观测信息应来自可靠数据，并记录来源与核对日期。
2. **文化分层**：现代星座、中国传统星官和其他文明的星空体系分别说明，不将它们简单视为一一对应关系。
3. **渐进升级**：先完善内容和数据结构，再引入复杂框架、实时计算或后端服务。
4. **移动优先**：星图、筛选和观测工具必须适合手机在户外使用。
5. **开放维护**：内容应拆分成独立数据和文章文件，方便通过 GitHub 持续修订与投稿。
6. **保护隐私**：定位功能保持可选，并支持手动选择城市或输入经纬度。

## 四、v0.2：八十八星座内容基础版

### 目标

一次性建立完整的 88 个现代星座目录，并为每个星座提供真实主要星位与连线、亮星、深空目标、观测路线、中国星官说明和西方神话或命名史。常见星座继续保持更丰富的观测内容，冷门与极南天星座也不保留空白条目。

### 首批重点深化星座

| 季节 | 星座 |
| --- | --- |
| 冬季 | 金牛座、双子座、大犬座、小犬座、御夫座 |
| 春季 | 狮子座、室女座、牧夫座、后发座 |
| 夏季 | 天鹰座、人马座、蛇夫座、武仙座 |
| 秋季 | 仙女座、英仙座、仙王座、双鱼座 |

### 每个星座的内容模板

1. 中文名、拉丁名、国际缩写和名称含义。
2. 最佳观测季节、时间、方向和适用纬度。
3. 主要恒星的名称、视星等、距离和光谱类型。
4. 真实恒星位置、传统连线和现代星座边界。
5. 代表星云、星团、星系和其他深空天体。
6. 中国传统星官及其与现代星座边界的差异。
7. 希腊及其他文明中的相关故事。
8. 裸眼、双筒望远镜和小型望远镜观测步骤。
9. 数据来源、参考资料和最后核对日期。

### 当前完成情况

- 已收录国际天文学联合会体系中的 88 个现代星座。
- 88 个星座均具备真实主要星位与连线、主要恒星和观测说明。
- 88 个星座均具备中国星官说明和西方神话或命名史。
- 已展示亮星、深空目标、数据来源和最后核对日期。
- 已支持按季节、可见范围、名称和收藏状态筛选。
- 数据文件具备自动完整性检查脚本。

## 五、v0.3：真实可探索星图

### 完成状态

`v0.3` 已完成。独立星座示意图已经升级为可连续移动的 Canvas 天球星图，并继续与 88 星座档案联动。

- 已支持鼠标滚轮、触摸板、触摸手势和屏幕控件缩放。
- 已支持拖动浏览相邻天区、复位全图和聚焦当前星座。
- 已加入赤经赤纬网格，以及星名、星座连线、现代星座边界和深空天体图层开关。
- 已依据视星等和 B−V 色指数区分恒星大小、亮度与颜色，并支持极限星等筛选。
- 已加入北天、赤道附近、南天和四季视图，用于快速整理和查找天区。
- 已支持点击恒星查看 Hipparcos 编号、J2000 坐标、日期坐标、视星等、色指数和 IAU 星座归属。
- 已采用 D3 处理地图投影与缩放交互，采用 Astronomy Engine 处理日期坐标转换和星座边界判定。
- 已将星图数据、绘图逻辑和页面内容拆分，保持纯静态部署能力。

### 功能范围

- 鼠标、触摸板和手机手势缩放。
- 拖动浏览相邻天区。
- 赤经赤纬坐标网格。
- 恒星亮度、颜色和大小差异。
- 星名、星座连线、边界和深空天体图层。
- 按极限星等筛选可见恒星。
- 点击恒星打开详细资料。
- 在相邻星座之间连续移动，而不是切换独立示意图。
- 支持北半球、南半球和不同季节视图。

### 技术原则

当前版本使用 Astronomy Engine 处理日期坐标转换和星座边界判定，使用 D3 处理天球投影与缩放交互，不自行编写这些核心算法。后续的升落与地平坐标计算继续沿用成熟天文计算库；视觉层和内容数据保持分离。

### 验收标准

- [x] 星图可以稳定缩放和拖动。
- [x] 星体位置来自统一的赤经赤纬数据。
- [x] 图层开关不会引发布局跳动。
- [x] 桌面端与触摸设备均可连续操作。

## 六、v0.4：今晚星空与观测助手

### 完成状态

`v0.4` 已完成并持续增强：支持 46 个地点预设、手动坐标、可选设备定位、完整 IANA 时区、日期时间、观测设备、地平星图、88 星座中心高度与升落近似、太阳暮光、月相月光、七大行星升落与过中天、精细化合月事件、当地可见性、整夜最佳时刻、光污染、可选天气和观测清单。天文计算在浏览器本地完成，天气查询默认关闭。

### 功能范围

- 手动选择城市或输入经纬度。
- 可选读取设备位置。
- 设置日期、当地时间和时区。
- 显示当前可见星座、方位和高度。
- 显示主要天体的升起、过中天和落下时间。
- 提供月相、月光影响和推荐观测窗口。
- 收录流星雨、行星合月等月度天象。
- 接入天气、云量或光污染信息时提供数据来源说明。
- 生成可复制或分享的观测清单。

### 验收标准

- [x] 不授权定位也能完整使用。
- [x] 时间和位置变化后，结果可以即时更新。
- [x] 观测结果明确标注日期、地点和时区。
- [x] 对中心位置近似、光污染估计、流星峰值日期和天气来源给出说明。
- [x] 区分极昼、持续天文黑夜和普通暮光窗口。
- [x] 天气发送位置前需要明确授权，并避免发送设备的完整精度坐标。
- [x] 根据裸眼、双筒或望远镜模式调整清单，并避免推荐超出设备能力的目标。
- [x] 分享链接可复现地点、时间、光污染和设备设置，同时对手动坐标降精度。
- [x] 结合整夜天文条件与逐小时天气给出推荐观测时刻。

### 世界星空专题

- 希腊英雄星座关系图。

### 内容规范

- 区分史料记载、后世传说和现代改编。

## 九、v1.0：完整中文星空图鉴

`v1.0` 的目标不是简单收齐 88 个星座名称，而是形成一套完整、可检索、可观测、可核查的内容体系。

### 建议发布标准

- 收录国际天文学联合会划分的 88 个现代星座。
- 收录主要亮星及常见中文、国际名称。
- 收录适合业余观测的代表深空天体。
- 提供真实天空位置和日期时间计算。
- 建立中国传统星官专题及跨文化故事资料库。
- 主要内容均有来源和最后核对日期。
- 完成桌面、平板、手机和无障碍测试。

## 十、建议的数据与内容结构

短期仍可使用原生 HTML、CSS 和 JavaScript，但应逐步将页面内容拆分出来：

```text
data/
  constellations/
  stars/
  deep-sky/
  observing/
content/
  stories/
  guides/
  history/
assets/
  images/
  charts/
```

`v0.7` 继续在现有原生静态技术栈上建立 E2E、数据 Schema、性能预算和迁移保护。通过详细计划定义的门槛后，`v0.8` 迁移到 `Astro + strict TypeScript`（内部使用 Vite），Canvas 星图和观测模块继续保留原生实现，不引入 React 作为默认前提。在内容和数据模型稳定之前，不优先建设账号、评论、私信或复杂后端。

## 十一、发布节奏

建议每两周发布一个小版本，每次更新至少完成以下四项中的三项：

- 深化或复核 3–5 个星座条目。
- 发布 1 篇星空文化专题。
- 完成 1 项星图或观测交互改进。
- 修正一批数据、无障碍或移动端问题。

每次发布在 GitHub Release 或更新日志中记录：新增内容、功能变化、数据修订、已知问题和下一版本目标。
